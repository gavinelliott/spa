State Pension Activation Prototype

Iteration 1

We’re testing this low impact approach for the first time. It includes options for offline paper prompt and direct online traffic. We’ve designed it targeting a specific customer group of single customers with a full NI Contribution because this specific customer group are the path of least resistance to work through a prototype when we don't have any technical solutions for the other customer groups.

  • Start Page

Iteration 1 - v2

A slightly different version which includes an element whereby the customer is prompted to wait for the delivery of their pension calculation. We’ve designed version 2 this way to test the assumption that customers will have a high expectation of getting their pension calculation online at the time of application and are looking to see their response.

  • Start Page

Iteration 2 - Alpha Phase

This iteration encompasses the whole MVP for Alpha. We've designed this iteration to test the assumption that customers will expect to actively get or defer their state pension as well as the dates they will receive their first few payments.

  • Start Page

Iteration 2 - v2

This iteration encompasses the user research analysis insights which we did with the first version. We're looking to use NI Number as an identifier only / reference only for a user. We've added in validation on form fields.

  • Start page

Iteration 3 - v1

This iteration takes in to account the feedback that we've had from our security colleagues, updating the 'what next' area using information from our user research, adding in a support channel and moving deferrals outside of the service. We've moved deferring outside of the service due to the result of the user research.
